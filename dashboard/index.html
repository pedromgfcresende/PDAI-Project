How It Works

Five specialised agents run in sequence. Each step hands off to the next, with a writer-critic loop at the end to ensure report quality before publishing.

Pulls papers & news from arXiv, RSS feeds, GitHub Trending, and Semantic Scholar.

Data Sources

Every day the system pulls from four public sources, normalises items into a common schema, and deduplicates before scoring. No API keys required for ingestion.

📄
arXiv
Preprint AI & ML research papers from cs.AI, cs.LG, and cs.CL — the primary signal for cutting-edge academic work before peer review.
📡 Daily pull  ·  Free  ·  No auth required
📰
RSS News Feeds
Industry news from TechCrunch AI, VentureBeat, Google AI Blog, Meta AI, OpenAI, and Anthropic. Captures product launches and company moves.
📡 Every few hours  ·  Free  ·  Standard RSS
💻
GitHub Trending
Trending AI repositories — a leading indicator of what the developer community is actually building and adopting right now.
📡 Daily  ·  Free  ·  Optional token for higher rate limits
🔬
Semantic Scholar
Paper metadata, citation counts, and influence scores. Helps identify which papers are gaining traction in the research community.
📡 Daily  ·  Free  ·  1000 req/sec, no auth
Quality Assurance

How Reports Are Scored

After the Synthesis Agent writes a draft, the Critic Agent (a different model family — Groq Llama) scores it across four dimensions. Using a different model prevents the writer's blind spots from passing review undetected.

⚓
Grounding
Are all claims traceable to the ingested source items? No hallucinated facts or unsupported assertions.
🧵
Coherence
Does the report flow logically? Are sections well-structured with clear transitions and no contradictions?
📦
Completeness
Does it cover the most significant developments from the period? No major stories missed.
🎯
Actionability
Does the reader walk away with clear strategic takeaways, not just a summary of what happened?
7+
Score ≥ 7.0 — Approved
Report meets quality bar and is published immediately.
<7
Score < 7.0 — Revised
Critic sends feedback to the writer for revision. Up to 2 revision attempts before force-publishing.
Getting Started

How to Use the Dashboard

Follow these steps to run the full pipeline locally and generate your first report.

1
Start the infrastructure
Open OrbStack or Docker Desktop, then start the PostgreSQL database.
docker compose up -d
2
Launch the API server
Start the FastAPI backend. Leave this terminal running — the dashboard connects to it on port 8000.
uv run uvicorn agent_service.main:app --reload --port 8000
3
Run the Daily Pipeline
Click the Run Daily Pipeline button on the Dashboard. This ingests items from all sources, scores them with the Filter Agent, and detects trend signals. Takes 1–3 minutes.
4
Generate a Report
Click Weekly Report or Monthly Report. Optionally set a custom date range first. The writer-critic pipeline runs and takes 30–60 seconds.
5
View & Download
Click any report card to open it. Use the HTML button to download a styled standalone file, or QMD to export a Quarto document you can render with quarto render.
